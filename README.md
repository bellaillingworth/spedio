# Spendio — Personal Spending Dashboard

> **Live demo:** [bpramjit.me](https://bpramjit.me) — try the dashboard, upload a sample CSV, or log a quick expense in your browser. No sign-up required.

## Description

Spendio is a lightweight personal finance dashboard aimed at college
students who want to understand where their money actually goes
without opening a spreadsheet. Users can log expenses manually, upload
a CSV bank statement, or drop in a PDF statement and have the rows
parsed and previewed in the browser before saving. Once saved, the
dashboard and history pages summarize spending with D3-powered charts
— totals, top category, day-by-day trends, and a monthly donut
breakdown. The app runs entirely on static files with a Supabase
backend; no build step is required. The app is currently deployed at
[bpramjit.me](https://bpramjit.me) on Azure Static Web Apps.

## Features

- Dashboard with four summary stat cards (total spent, top category,
  transaction count, average per day) for the current month
- Teal hero bar chart showing daily spending across the month, with
  keyboard-focusable bars and screen-reader-friendly labels
- CSV bank statement upload parsed entirely in the browser, with a
  preview table that lets you recategorize, edit descriptions,
  exclude likely credits, or remove any row before saving
- PDF bank statement upload via `pdf.js`, with statement-period
  detection so bare `MM/DD` transactions get the correct year — even
  for statements that wrap from December into January
- Manual entry form with native date picker, category select, and
  validated amount input
- Spending history browser with prev/next month navigation, a
  jump-to-month select, and a donut + line-chart view for any month
  in the dataset
- Fully responsive layout from 320px and up — sidebar collapses to a
  horizontal nav on small screens
- WCAG 2.1 Level AA accessibility throughout (semantic landmarks,
  skip link, focus-visible rings, labeled form controls, contrast
  audited design tokens, keyboard-navigable charts)
- Honors `prefers-reduced-motion`
- Optional deploy via the included Azure Static Web Apps config

## Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) (v18 or higher) — required to run `npx serve`
- A modern web browser (Chrome, Firefox, Safari, or Edge)
- A [Supabase](https://supabase.com/) project (free tier is fine) if
  you want saves to persist; the app runs read-only without one and
  surfaces a yellow "Supabase is not configured" banner
- A terminal / command line

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/bellaillingworth/spedio.git
   ```

2. Navigate into the project folder:
   ```bash
   cd spedio
   ```

3. Create the Supabase table. In the
   [Supabase SQL editor](https://supabase.com/dashboard), paste and
   run the contents of `db/schema.sql`. That creates the
   `transactions` table, indexes, and permissive row-level-security
   policies for single-user anonymous use.

4. Configure credentials. Open `js/supabase-client.js` and replace
   the two placeholder constants with the Project URL and anon key
   from your Supabase dashboard:
   ```js
   const DEFAULT_SUPABASE_URL = "https://YOUR-PROJECT.supabase.co";
   const DEFAULT_SUPABASE_ANON_KEY = "eyJhbGciOi...";
   ```
   Alternatively, set `window.SPENDIO_CONFIG` in an inline script at
   the top of each HTML page before the module scripts load.

5. Start the development server:
   ```bash
   npx serve . -l 5000
   ```

6. Open your browser and visit:
   ```
   http://localhost:5000
   ```

## Usage

Once the server is running, open `http://localhost:5000` in your
browser. Use the left sidebar to move between the four pages:

- **Home** — This month's dashboard with totals and the daily hero
  chart.
- **Upload** — Drop in a CSV or PDF bank statement, review the
  parsed preview table, recategorize or exclude rows, then confirm
  to save.
- **Add Expense** — Log a single transaction with a date, category,
  optional description, and amount.
- **History** — Browse any prior month with the ← / → buttons or the
  jump-to-month dropdown; see a category donut and daily line chart
  for the selected month.

To customize content, edit the HTML files directly. All text lives in
`index.html`, `upload.html`, `manual-entry.html`, and `history.html`.
To change colors, typography, or spacing, edit the CSS custom
properties under `:root` in `css/styles.css` — the D3 charts read
these tokens at runtime, so the whole app restyles from one place.

## Known Issues / To-Do

- [ ] No edit or delete affordance on saved transactions — only add
  and import. History tables should eventually expose a remove
  button.
- [ ] Currency is hard-coded to USD (`$`, `"Amount (USD)"`,
  `formatCurrency`). No locale config yet.
- [ ] Scanned / image-only PDFs are unsupported — the parser relies
  on real text content. The UI surfaces a specific error but there
  is no OCR fallback.
- [ ] No favicon configured — browser tab shows a blank icon.
- [ ] No `LICENSE` file committed yet, even though the README and
  badge reference MIT.
- [ ] Placeholder "Student" user identity everywhere; needs real
  auth (and a `user_id` column + tighter RLS) before going
  multi-user.

## Roadmap

- Add a delete / edit row affordance to the History and preview
  tables
- Budget per category with an "over budget" indicator on the
  dashboard
- CSV export of the full transaction history
- Dark-mode palette driven by a single `data-theme` switch on
  `<html>`
- Multi-user support with real authentication (Supabase Auth or
  magic-link email), a `user_id` column on `transactions`, and
  tightened RLS policies
- Locale / currency config via `window.SPENDIO_CONFIG`
- Favicon, Open Graph meta tags, and a small hero animation for
  polish
- Optional OCR pass for scanned PDF statements

## Contributing

Spendio is a personal project, but bug reports, accessibility
improvements, and design suggestions are welcome. To contribute:

1. Fork the repository on GitHub
2. Create a new branch for your change:
   ```bash
   git checkout -b your-feature-name
   ```
3. Make your edits to the HTML, `css/styles.css`, or the relevant
   module under `js/`
4. Run the page through the W3C HTML and CSS validators and a
   contrast/accessibility checker (axe DevTools or Lighthouse)
   before opening a PR — see the targets listed in `WORKING_NOTES.md`
5. Commit your changes with a clear message:
   ```bash
   git commit -m "Describe what you changed"
   ```
6. Push the branch to your fork and open a pull request against
   `main`
